## 倒角 Bug 修复

- **问题**：拖动倒角滑块盒子消失
- **原因**：`rebuildGeometry()` 销毁旧材质后重建，纹理引用（texCache）虽然未 dispose，但 WebGL 纹理资源在新材质绑定后可能出现渲染异常
- **修复方案**：`rebuildGeometry()` 改为只替换几何体（`boxMesh.geometry = newGeo`），不触碰材质和纹理。Edges 也只替换 geometry。这样完全避免材质/纹理生命周期问题
- **关键认知**：BoxGeometry 和 RoundedBoxGeometry 都有 6 个 material groups（顺序一致），所以交换几何体后材质数组仍然正确映射到各面
- **代码简化**：rebuildGeometry 从 ~25 行缩减到 ~10 行，不再需要 dispose/recreate 材质

### 问题
- 用户反馈"水族灯_SR-616显示不出来"
- 排查发现 `boxes/水族灯_SR-616.lnk` 是 Windows 快捷方式文件（.lnk），不是真正的目录
- HTTP 服务器（Python http.server）无法解析 .lnk 文件，浏览器请求 `boxes/水族灯_SR-616/front.png` 返回 404

### 解决方案
1. 删除 `.lnk` 文件
2. 使用 `mklink /J` 创建目录联接（junction），无需管理员权限
   - 命令：`mklink /J "C:\Users\Man-202403\.qclaw\workspace-agent-75527ef5\boxes\水族灯_SR-616" "F:\1-文档-包装\Seaoura-通用包装盒\boxes\水族灯_SR-616"`
3. 验证：联接成功，目录下 6 张 PNG 文件可见

### 关键认知
- **Windows .lnk 文件 vs symlink/junction**：
  - `.lnk` 是 Shell 快捷方式，只有 Windows 资源管理器能解析，HTTP 服务器不行
  - 符号链接（symlink）和目录联接（junction）是文件系统级别的，所有程序都能访问
  - `mklink /J` 创建联接不需要管理员权限，`mklink /D` 创建符号链接需要
- 3D 纹理加载路径必须是 HTTP 服务器可访问的真实路径或文件系统链接

### 解决方案
1. 删除 `.lnk` 文件
2. 使用 `mklink /J` 创建目录联接（junction），无需管理员权限
   - 命令：`mklink /J "C:\Users\Man-202403\.qclaw\workspace-agent-75527ef5\boxes\水族灯_SR-616" "F:\1-文档-包装\Seaoura-通用包装盒\boxes\水族灯_SR-616"`

### 关键认知
- **Windows .lnk 文件 vs symlink/junction**：
  - `.lnk` 是 Shell 快捷方式，只有 Windows 资源管理器能解析，HTTP 服务器不行
  - 符号链接（symlink）和目录联接（junction）是文件系统级别的，所有程序都能访问
  - `mklink /J` 创建联接不需要管理员权限，`mklink /D` 创建符号链接需要
- 3D 纹理加载路径必须是 HTTP 服务器可访问的真实路径或文件系统链接
